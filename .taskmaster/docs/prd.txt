# Overview  
이 프로젝트는 학생들이 직접 만든 웹사이트를 온라인에서 전시할 수 있는 갤러리 웹 애플리케이션입니다. 학생들은 자신의 웹사이트를 제출하고, 방문자들은 다양한 학생 작품을 탐색하고 감상할 수 있습니다. 이 서비스는 학생들의 창의성과 개발 역량을 널리 알리고, 서로의 작업에서 영감을 얻을 수 있는 장을 제공합니다.

# Core Features  
- 학생 웹사이트 등록: 학생들이 자신의 웹사이트(링크, 썸네일, 설명 등)를 등록할 수 있습니다.
- 갤러리 뷰: 제출된 웹사이트들이 썸네일과 함께 그리드 형태로 전시됩니다.
- 상세 페이지: 각 웹사이트의 상세 정보(설명, 제작자, 기술 스택 등)를 볼 수 있습니다.
- 검색 및 필터: 키워드, 제작자, 기술 스택 등으로 작품을 검색/필터할 수 있습니다.
- 반응형 UI: 데스크탑, 태블릿, 모바일에서 모두 쾌적하게 이용할 수 있습니다.

# User Experience  
- **학생**: 로그인 없이 웹사이트를 등록(간단한 폼 입력) → 등록 후 갤러리에서 자신의 작품 확인
- **방문자**: 갤러리에서 다양한 학생 웹사이트 탐색 → 썸네일 클릭 시 상세 정보 확인 → 외부 링크로 실제 사이트 방문 가능
- UI/UX: 직관적이고 시각적으로 매력적인 그리드 레이아웃, Tailwind 기반의 깔끔한 디자인, 빠른 탐색 경험

# Technical Architecture  
- **Frontend**: React + Vite + TailwindCSS
- **State 관리**: React Context 또는 간단한 상태 관리 라이브러리
- **데이터 저장**: 초기 버전은 로컬 JSON(Mock Data) 사용, 추후 Firebase 등 백엔드 연동 고려
- **컴포넌트 구조**: 갤러리(Grid), 등록 폼, 상세 모달/페이지, 검색/필터 바 등
- **배포**: Vercel, Netlify 등 정적 호스팅 서비스 활용

# Development Roadmap  
- **MVP**
  - 갤러리 뷰(그리드) 구현
  - 학생 웹사이트 등록 폼(간단한 입력, 썸네일 업로드는 URL 기반)
  - 상세 정보 모달/페이지
  - Mock 데이터 기반 동작
  - 반응형 레이아웃
- **향후 개선**
  - 실시간 데이터 저장(백엔드 연동)
  - 소셜 공유 기능
  - 좋아요/댓글 등 인터랙션
  - 관리자 기능(작품 승인 등)

# Logical Dependency Chain
- UI 프레임(레이아웃, 네비게이션) → 갤러리 그리드 → 등록 폼 → 상세 페이지/모달 → 검색/필터 → 반응형 → 데이터 연동(추후)

# Risks and Mitigations  
- 썸네일 이미지 관리(외부 링크만 허용, 용량 제한)
- 데이터 유실(초기엔 Mock, 추후 백엔드 연동 필요)
- 저작권/부적절한 콘텐츠(관리자 승인 기능으로 대응)
